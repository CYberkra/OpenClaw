# 2026-03-13

- 规则纠正：此前对“子分区/频道/类别”的解释有误，已废弃。
- 最新定义（以用户本次最新指令为准）：
  - “子区” = Discord 线程（Thread），用于处理短期事项/很快解决的问题；通常挂在某个频道下，处理完后不再长期保留。
  - “频道” = Discord 频道（Channel），用于处理长期事项；长期持续存在，用于稳定跟进。
  - “类别” = Discord 类别（Category），用于承载不同大项目/复杂事项的更高层级结构；其下可包含多个频道。
- 规划原则：
  - 短期问题 → 放子分区
  - 长期事项 → 放频道
  - 大项目/复杂项目 → 放类别
- 之前基于错误理解创建的“自然基金申请书一次性子任务方案”和错误类别结构视为废案，不再作为术语执行标准。
- 用户新增规则：之后回复不需要再附带 token 消耗（取消每条回复末尾 token in/out 汇报）。
