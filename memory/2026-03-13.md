# 2026-03-13

- 规则纠正：此前对“子分区/频道/类别”的解释有误，已废弃。
- 最新定义（以用户本次最新指令为准）：
  - “子分区” = Discord 中用于处理短期事项/很快解决的问题的结构。特点：短期、一次性、处理完后通常不再长期保留。
  - “频道” = Discord 中用于处理长期事项的结构。特点：长期持续存在，用于稳定跟进。
  - “类别” = Discord 中用于承载不同大项目/复杂事项的更高层级结构。特点：项目级、复杂度高、可长期保留，其下可包含多个频道/子分区。
- 规划原则：
  - 短期问题 → 放子分区
  - 长期事项 → 放频道
  - 大项目/复杂项目 → 放类别
- 之前基于错误理解创建的“自然基金申请书一次性子任务方案”和错误类别结构视为废案，不再作为术语执行标准。
