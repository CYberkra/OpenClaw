# 组会PPT固定工作流

## 长期流程

### A. 新建一期汇报

1. 复制 `BRIEF_TEMPLATE.md`
2. 填写本次组会目标、听众、页数、重点结果
3. 把材料放入：
   - 图片 → `assets/`
   - 数据 → `data/`
   - 备注 → `notes/`
4. 在 `deck.md` 生成本期主稿

### B. 制作顺序

1. **定结构**：先决定 5-8 页还是 8-12 页
2. **定主线**：本次到底讲“进展、结果、问题、计划”中的哪个为主
3. **放证据**：每页必须有图/表/数字/结论，不写空话
4. **统一视觉**：标题、配色、卡片样式统一
5. **导出检查**：检查字数、图清晰度、逻辑跳跃、页数

### C. 每次汇报结束后

1. 将本期版本复制到 `history/YYYY-MM-DD_topic/`
2. 保存：
   - 使用过的 `deck.md`
   - 导出 HTML/PDF/PPTX
   - 关键图片与结论摘要
3. 在 `notes/` 留下：
   - 老师/组会反馈
   - 下次必须回应的问题

## 适用的四种组会模板

### 1. 进展型
- 我这周做了什么
- 和上周相比推进了什么
- 当前卡点
- 下周计划

### 2. 结果型
- 问题定义
- 方法/配置
- 核心结果图
- 对比与解释
- 结论

### 3. 问题诊断型
- 遇到的问题
- 怀疑原因
- 验证过程
- 目前判断
- 下一步修复方案

### 4. 汇总答辩型
- 研究背景
- 总体方法
- 多阶段结果
- 创新点/亮点
- 结论与计划

## 页面质量标准

每页至少满足下面之一：

- 有一张能说明问题的图
- 有一个关键数字/指标
- 有一个明确结论
- 有一项对比（before/after、A/B、方法对比）

如果一页只有大段文字，通常说明这一页还没准备好。

## 默认排版策略

- 标题页：深色背景，强识别度
- 内容页：浅底 + 卡片化布局
- 结果页：大图优先，文字少而准
- 结论页：3 条以内结论 + 3 条以内下一步

## 默认响应协议

以后在这个线程里只要你说：

- “做这周组会PPT”
- “更新上次那份组会汇报”
- “把这几张图整理成组会 slides”

我会默认进入这个子分区工作流：

1. 先整理内容结构
2. 再更新 `deck.md`
3. 再导出 HTML 版本
4. 需要时再转 PPT/PDF
