# GPR组会实战版｜讲稿与答疑（终版）

## 一页讲稿（3-5分钟）
1. 本次汇报只讲项目实跑，不讲泛化科普：数据是 `A8-NEW-1.csv`，证据来自 GUI 实际运行目录。
2. 目标是解决两件事：
   - 压制水平背景条纹；
   - 提升深层弱反射可见性，同时保持结构真实性。
3. 推荐默认链路：`time-zero → dewow → SVD background → F-K → SEC → Hankel-SVD`。
4. 关键结论：
   - SVD 背景抑制比平均背景更稳健，条纹残留更少；
   - SEC 比 AGC 更保结构；
   - AGC 在当前样本易过均衡，导致相对振幅信息衰减。
5. 当前缺口：GUI 暂未暴露 `bandpass` 与 `wavelet`。
6. 下一步：补齐 bandpass/wavelet、增加多样本定量指标（SNR/结构相似/可分性）、固化参数模板批处理复跑。

## 答疑页（高频问题）
### Q1：为什么默认推荐 SVD 而不是 subtract_avg？
A：同样数据下，SVD 的背景压制更稳定，对目标形态干扰更小；平均背景在部分区段仍有条纹残留。

### Q2：为什么 SEC 默认优于 AGC？
A：AGC 在当前样本出现明显过均衡，容易把全局对比“拉平”；SEC 的增益更可控，结构连续性保留更好。

### Q3：这套流程可复现吗？
A：可复现。发布包内 `evidence/summary.json` 包含步骤、参数、耗时与快照路径；HTML 页面内证据图均来自同一实跑目录。

### Q4：当前最大的工程限制是什么？
A：GUI 方法列表里 bandpass/wavelet 尚未暴露，频带细控与小波去噪暂时无法直接走 GUI 方案。

### Q5：如果必须用 AGC，怎么降低风险？
A：仅在“弱反射补偿试验”场景启用，缩小窗口/降低增益强度并保留 A 链作为对照，避免直接替换默认流程。
