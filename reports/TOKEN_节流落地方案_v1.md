# TOKEN 节流落地方案 v1（不降质）

## 目标
在保证 GPR 交付质量的前提下，降低 token 消耗与上下文膨胀风险。

## 执行规则（立即生效）
1. 固定前缀：系统约束/项目固定路径/输出格式保持稳定顺序，提升缓存命中。
2. 最小上下文：仅注入“当前任务必要文件 + 当天关键记忆”；长历史改摘要引用。
3. 分级执行：
   - 复杂任务（代码改造/多步骤验证）→ 子代理
   - 简单问答/状态更新 → 低 thinking + 短回复
4. 汇报压缩：统一“结论 + 证据路径 + 下一步”，避免贴长日志。
5. Token回报：每条回复附本次 token（in/out）。

## 适配当前工作流
- 主战场：Discord
- FIS：暂停
- PPT：默认 html-presentation
- GPR：证据优先（路径 + 命令 + 关键输出）

## 验收
- 会话缓存命中率维持高位
- 无关键信息遗漏导致返工
- 用户感知响应速度与质量不下降
